#  Lab 4: DHCP Configuration on Router

> **Tools Used:** Cisco Packet Tracer / GNS3  
> **Focus:** Configuring DHCP on a router to dynamically assign IP addresses 

---

## 🎯 Objectives

- Understand the role of **DHCP** in networks
    
- Configure a router as a **DHCP server**
    
- Assign IP addresses dynamically to multiple devices
    
- Exclude specific IP addresses from DHCP
    
- Verify DHCP functionality using `ping` and device network settings
    

---

## 🔹 1️⃣ Introduction

- Previously, IPs were **statically assigned** manually
    
- In large networks, manual assignment is impractical
    
- **DHCP (Dynamic Host Configuration Protocol)** automates IP assignment
    

---

## 🔹 2️⃣ DHCP Scenarios

1. **Dedicated DHCP Server**
    
    - Separate server runs DHCP service
        
    - Preferred in large networks to reduce router load
        
2. **Router as DHCP Server**
    
    - Router itself assigns IPs
        
    - Suitable for labs or small networks
        

---

## 🔹 3️⃣ Configuring DHCP on the Router

### Step 1: Enter CLI

```lua
Router> enable  
Router# configure terminal
```
### Step 2: Create DHCP Pool

```lua
Router(config)# ip dhcp pool POOL1
```

- `POOL1`: any name (letters/numbers)
    

### Step 3: Specify Network and Subnet Mask

```lua
Router(dhcp-config)# network 192.168.1.0 255.255.255.0
```

### Step 4: Set Default Gateway

```lua
Router(dhcp-config)# default-router 192.168.1.1
```

### Step 5: Set DNS Server

```lua
Router(dhcp-config)# dns-server 8.8.8.8
```

- Can be external (Google) or internal (Domain Controller)
    

```lua
Router(dhcp-config)# dns-server 192.168.1.1
```

---

## 🔹 4️⃣ Exclude IP Addresses

- Prevent certain IPs from being assigned (e.g., servers, printers)
    

```lua
Router(config)# ip dhcp excluded-address 192.168.1.10 192.168.1.20
```

- Can exclude a **range** or **single IP**
    

---

## 🔹 5️⃣ Verify DHCP Operation

1. Save configuration:
    

```lua
Router# copy running-config startup-config
```

2. On PCs: select **Obtain IP Automatically**
    
3. Devices should receive:
    
    - IP Address ✅
        
    - Subnet Mask ✅
        
    - Default Gateway ✅
        
    - DNS ✅
        

---

## 🔹 6️⃣ Adding DHCP Pool for Another Network

```lua
Router(config)# ip dhcp pool POOL2  
Router(dhcp-config)# network 192.168.2.0 255.255.255.0  
Router(dhcp-config)# default-router 192.168.2.1  
Router(dhcp-config)# dns-server 8.8.8.8
```

---

## 🔹 7️⃣ Test Connectivity

- Ping between devices → should succeed ✅
    
- Ping to router → should succeed ✅
    

```lua
Router# ping 192.168.1.2
```

- Successful response shown with `!!!!`
    

---

## 🔹 8️⃣ Important Notes

- Large networks → use **dedicated DHCP Server**, not router
    
- Can create multiple pools for multiple networks
    
- Servers and printers should have **Static IPs** for stability
    

---

## 🔹 9️⃣ Removing DHCP Pool

```lua
Router(config)# no ip dhcp pool POOL1
```

---

## 🔑 Summary

- DHCP saves time and effort in assigning IP addresses
    
- In small labs: router as DHCP server is fine
    
- In large environments: use dedicated DHCP server
    
- Always exclude IPs for critical devices (servers/printers)
