#  Lab 5: Static Routing Configuration

> **Tools Used:** Cisco Packet Tracer / GNS3  
> **Focus:** Configuring Static Routes between multiple routers to enable connectivity between distant networks 

---

## 🎯 Objectives

- Understand the difference between **Static** and **Dynamic Routing**
    
- Configure **Static Routes** on routers
    
- Enable connectivity between remote networks
    
- Test connectivity using `ping` and `tracert`
    

---

## 🔹 1️⃣ Introduction

- Router's main function: **connect different networks**
    
- Single router → sees only **Connected networks**, no routing needed
    
- Multiple routers → need **Routing** to reach distant networks
    

---

## 🔹 2️⃣ Routing Types

### 1. Static Routing

- **Manual configuration** by the administrator
    
- Suitable for **small networks (2–4 routers)**
    
- ✅ Advantages:
    
    - No overhead on router
        
    - More secure (manual control)
        
- ❌ Disadvantages:
    
    - High administrative effort
        
    - Must know all network details
        
    - Not practical for large networks
        

### 2. Dynamic Routing

- Routers automatically exchange routes using protocols like **RIP, OSPF**
    
- ✅ Advantages: Easy, automatic, flexible
    
- ❌ Disadvantages: Consumes router resources (CPU, RAM)
    

---

## 🔹 3️⃣ Lab Scenario

- Two branches: **Cairo** & **Alexandria**
    
- Each branch has: **LAN + Router**
    
- Point-to-Point link between routers
    
- Before routing:
    
    - Each router sees only its connected networks
        
    - PCs in Cairo cannot reach Alexandria and vice versa
        

---

## 🔹 4️⃣ Lab Goal

- Enable each router to know about **remote networks** using **Static Routes**
    

---

## 🔹 5️⃣ Static Route Command Format

```lua
Router(config)# ip route [Destination_Network] [Subnet_Mask] [Next_Hop_IP or Exit_Interface]
```

- **Destination_Network** → target network
    
- **Subnet_Mask** → network mask
    
- **Next_Hop_IP** → router IP towards target network
    
- Or use **Exit_Interface** if directly connected
    

---

## 🔹 6️⃣ Practical Example

### (A) Cairo Router

- Connected Networks:
    
    - 192.168.1.0/24 (LAN)
        
    - 10.0.0.0/30 (Link to Alexandria)
        
- Needs to reach: 200.200.0.0/24 (Alexandria LAN)
    

```lua
RouterCairo(config)# ip route 200.200.0.0 255.255.255.0 10.0.0.2  
RouterCairo# do show ip route
```
### (B) Alexandria Router

- Connected Networks:
    
    - 200.200.0.0/24 (LAN)
        
    - 10.0.0.0/30 (Link to Cairo)
        
- Needs to reach: 192.168.1.0/24 (Cairo LAN)
    

```lua
RouterAlex(config)# ip route 192.168.1.0 255.255.255.0 G0/1   # using Exit Interface
```

---

## 🔹 7️⃣ Testing Connectivity

```lua
C:\> tracert 10.10.10.3   
C:\> ping 10.10.10.3
```
- Should succeed if static routes are correct ✅
    

---

## 🔹 8️⃣ Removing a Static Route

```lua
Router(config)# no ip route [Destination_Network] [Subnet_Mask] [Next_Hop_IP or Exit_Interface]
```

---

## 🔹 9️⃣ Important Notes

- Next Hop can be specified as:
    
    - **IP Address** (recommended)
        
    - **Exit Interface** (with conditions)
        
- More routers → manual configuration becomes harder → use **Dynamic Routing**
    
- Always assign IPs on diagrams **before lab execution (best practice)**
    
- Can configure **Backup Route (Floating Static Route)** for redundancy
    

---

## 🔑 Summary

- **Static Route:** manual, secure, suitable for small networks, high admin effort
    
- **Dynamic Route:** automatic route exchange, suitable for large networks, less admin effort
    
- Command format:
    

```lua
ip route [Destination] [Mask] [Next-Hop]
```
