## 🔹 7. Testing Connectivity

### Basic Commands

- `ipconfig` → Shows IP, Subnet, Gateway
    
- `ping <IP>` → Test connectivity
    

### Execution

- Open CMD on the device
    
- Run:
    
    - `ipconfig` to verify IP
        
    - `ping <target IP>` to check connectivity
        

✅ **Reply from…** → Connection successful  
❌ **Request timed out** → Connection failed

---

## 🔹 8. `ipconfig /all` Explanation

- Shows complete network interface details:
    
    - IP Address, Subnet Mask, Gateway, DNS, MAC, DHCP status
        
- Helps verify correct configuration
    
- Detects auto-configured addresses (169.254.x.x → no DHCP)
    

---

## 🔹 9. Ping Output Explanation

```sh
C:\>ping 192.168.1.70  
  
Pinging 192.168.1.70 with 32 bytes of data:  
  
Reply from 192.168.1.70: bytes=32 time<1ms TTL=128  
Reply from 192.168.1.70: bytes=32 time<1ms TTL=128  
Reply from 192.168.1.70: bytes=32 time=9ms TTL=128
```
### Interpretation

- `Request timed out` → Initial packet lost (ARP request or device initializing)
    
- `Reply from ...` → Successful connection
    
- Statistics: 4 sent, 4 received → 0% packet loss
    

---

## 🔹 10. Ping for Troubleshooting

1. **100% loss** → Check IP, cable, switch port, firewall
    
2. **First packet timeout only** → Usually ARP request
    
3. **Ping Gateway** → Check local network vs external
    
4. **Ping by DNS name** → Test DNS resolution
    

---

## 🔹 11. Firewall Considerations

- Firewalls can block ICMP requests
    
- May see no reply despite devices being online
    
- Solution: Add **ICMP exception** in firewall
    

---

## 🔹 12. Advanced Ping Options

- `ping -l <size>` → Change packet size
    
- `ping -t` → Continuous ping for stability testing
    

⚠️ Warning: Excessive packet size or continuous ping can cause network issues (DoS)

---

## 🔹 13. Quick Reference Table

|Ping Result|Possible Cause|Solution|
|---|---|---|
|`Reply from <IP>`|Normal connectivity|✅|
|`Request timed out.`|Device off / firewall / misconfig|Check device, IP, firewall|
|`Destination host unreachable.`|No route to target|Check subnet, gateway, routing table|
|`Ping: transmit failed. General failure.`|TCP/IP stack issue|Restart adapter, `ipconfig /renew`|
|`Reply from <Gateway IP>: Destination host unreachable.`|Router knows target not exist|Check routing, IP assignment|
|High latency (>200ms)|Network congestion|Test another network, check speed, use `tracert`|
